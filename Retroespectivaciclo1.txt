1. ¿Cuáles fueron los mini-ciclos definidos? Justifíquenlos.
Definimos 3 mini-ciclos:

Mini-ciclo 1: Estructura base
        Crear la clase StackItem con sus atributos (type, number, color, height)
        Crear la clase Tower con el constructor y el ArrayList de items
        Justificación: Sin estas clases no se puede hacer nada. Es la base de todo el proyecto.

Mini-ciclo 2: Funcionalidad de tazas y tapas
        Implementar pushCup, popCup, removeCup
        Implementar pushLid, popLid, removeLid
        Implementar las validaciones (que no se repitan, que quepan, etc.)
        Implementar height() y ok()
        Justificación: Son los requisitos funcionales principales. Sin poder agregar y quitar elementos el simulador no tiene sentido.

Mini-ciclo 3: Representación visual
        Implementar el dibujo de las tazas como forma de "U" usando Rectangles
        Implementar el dibujo de las tapas como rectángulos sólidos
        Implementar makeVisible, makeInvisible y redraw
        Dibujar la línea base de la torre
        Justificación: Los requisitos de usabilidad exigen representación visual. Lo dejamos de último porque el simulador debe funcionar primero en modo invisible.

4. ¿Cuál consideran fue el mayor logro? ¿Por qué?

    El mayor logro fue lograr que la representación visual funcionara correctamente usando solo los métodos de shapes. El paquete shapes no tiene un método moveTo para posicionar un Rectangle en una coordenada absoluta, solo tiene moveHorizontal y moveVertical que son relativos. Tuvimos que diseñar una estrategia donde guardamos la posición actual de cada Rectangle y calculamos el desplazamiento necesario para moverlo a donde queremos.

5. ¿Cuál consideran que fue el mayor problema técnico? ¿Qué hicieron para resolverlo?

    El mayor problema técnico fue el posicionamiento de los elementos visuales. Como los Rectangles de shapes se crean con una posición fija (x=70, y=15) y solo se pueden mover con distancias relativas, al principio los elementos aparecían en posiciones incorrectas o se superponían.

    Para resolverlo creamos un método privado mover que recibe la posición actual y la posición destino, calcula la diferencia y usa moveHorizontal y moveVertical con esa diferencia. Además guardamos siempre las coordenadas actuales de cada Rectangle en variables como baseX, baseY.
